# Changelog
## [0.13.0] - 2026-03-20
### 追加
- クラウドストレージ対応を拡張: OneDrive（個人・法人）/ Google Drive for Desktop を `path_aliases` で利用可能に
  - `%OneDrive%`、`%OneDriveCommercial%` 環境変数によるパス解決に対応
  - Google Drive はドライブレターを `path_aliases` で定義することで利用可能
- README にクラウドストレージ別の設定例・事前準備手順を追記

## [0.12.1] - 2026-03-19
### 修正 (qgis_launcher)
- `qgis_settings.json` 読み込み時にバックスラッシュを自動修正する処理を追加
  - Windows のエクスプローラからコピーした単一バックスラッシュのパス（例: `"C:\ProjectFiles2"`）を含む JSON でも正常に読み込めるよう `fix_backslashes_in_json()` 関数を追加
  - 読み込み時に常に事前修正してからパースする方式を採用（フォールバックではなく確実に変換）
  - 有効な JSON エスケープ（`\"`, `\\`, `\/`, `\b`, `\f`, `\n`, `\r`, `\t`, `\u`）はそのまま保持し、それ以外の `\` のみ `\\` に変換
  - 混在パス（例: `"C:\github\\ReEarth_Portal\qgis_launcher"` ）も正しく処理

## [0.12.0] - 2026-03-19
### 変更 (qgis_launcher)
- プロジェクト候補リストの生成ロジックを刷新
  - `project_path` の各エントリを**拡張子で先にファイル/フォルダを判定**し、一覧を構築するよう変更
  - ファイル指定（`.qgs` / `.qgz`）: ファイルが存在すれば 1 エントリとして追加
  - フォルダ指定: 直下にある `.qgs` / `.qgz` を列挙して追加
  - パス解決はすべて `settings_dir` 基準で行い、絶対パスはそのまま優先使用
- GUI の表示名を常に **「直近1階層上のフォルダ名 - ファイル名」** 形式に統一
  - ルート直下ファイル（例: `C:\file.qgs`）はドライブ文字を親名として表示（例: `C: - file.qgs`）
  - 内部保持・保存・起動はすべて実絶対パスで行うため動作への影響なし
- `display_name_for()` ヘルパー関数を追加し、表示名生成ロジックを一元化

## [0.11.0] - 2026-03-15
### 追加・改善 (qgis_launcher)
- インストール版、ポータブル版など複数のQGISバージョンおよびQFieldの自動検出機能を追加
- ランチャーGUIに「QGIS Version」の選択用プルダウンメニューを追加
- コマンドライン実行時に `--qgis_executable` 引数で起動するQGISの実行ファイルパスを直接指定できる機能を追加
- 選択されたQGISの実行ファイルパス（`qgis_executable`）を `qgis_settings.json` に保存・復元する機能を追加
## [0.10.0] - 2026-03-15
- ランチャーの大幅なUI統合とリファクタリングを実施
  - `qgis_launcher.exe` 単体でGUIが起動するように統合 (FLTK ライブラリをバンドルビルドで静的リンク)
  - 起動オプション `--cli` を追加し、CUI専用モードのフォールバック動作を維持
  - プロファイルとプロジェクトの入力欄を「手入力可能コンボボックス (ドロップダウン)」へ変更
  - `Settings Dir` 配下のディレクトリから既存の `.qgs` / `.qgz` を自動取得してドロップダウン候補に追加する機能を追加
  - サブフォルダ内の同名プロジェクトファイル選択時の不具合（FLTKのパス区切り文字誤認など）を修正・対応
  - `%APPDATA%` 内に存在する既存の QGIS プロファイルディレクトリも自動でリストの選択肢に加えるよう探索ロジックを拡張
  - GUIを整理し、不要になった `Load` `Save` ボタンおよび `Settings Dir` の画面入力を削除してスッキリとした起動特化UIにリデザイン

## [0.9.0] - 2026-03-15
- リポジトリクリーンアップ: ルートのビルド出力（`.next`, `node_modules`, `out`, `public`）および `qgis_launcher/target` を削除

 - ランチャー: デフォルトで GUI を起動するように変更（`gui_fltk` バイナリを追加）。

## [0.8.0] - 2026-03-14
- ランチャー: サーバーモードを削除し、起動ロジックを簡素化
- 起動時の設定優先順を `qgis_settings.json` を第一に変更（CLI はフォールバック）
- スタートアップ登録機能を削除
- プロファイルのコピー処理を安全化: 既存プロファイルは削除せず、不足分のみコピーする挙動に変更
- プロジェクト選択ロジックを強化: `project_path` がファイル名のみの場合、実行ファイル隣の「プロジェクト名」フォルダ内のファイルを優先して探索するよう変更
- 起動時に実行ファイルパス / カレントディレクトリをデバッグログ出力するよう追加
- いくつかの未使用コードを削除（コードクリーンアップ）
- README を統合・更新

## [0.7.0] - 2026-03-14
- フロントエンドの設定読み込み/初期化を `portal:settings` へ一本化
  - `src/PortalContext.js` の初期化ロジックを統一し、全ての設定値（Re:Earth, QGIS, launcherDirなど）を `portal:settings` から読み込むように変更
  - レガシーな個別キー保存（per-key storage）を廃止し、`resetTo` は `portal:settings` を削除する動作へ統一
- UI の整理: BOX タブと BOX 向け設定パネルを削除
  - `pages/index.js` から BOX タブ・iframe・設定セクションを削除して、関連 state 参照を整理
- その他の修正
  - ローカルランチャー呼び出しのフロント側統合を削除し、ブラウザ側は localStorage / File System Access API にフォールバックするよう整理

## [0.6.0] - 2026-03-14
- ブラウザ側でのファイル入出力を改善
  - `設定ファイルを読み込む` を `showOpenFilePicker`（対応ブラウザ）で直接ファイル選択して読み込む挙動に変更。非対応時は既存のファイル入力にフォールバック。 (`pages/index.js`, `src/PortalContext.js`)
  - ブラウザで `qgis_settings.json` を保存できる `saveToFs` を追加し、UI上からファイルとして保存できるようにした（File System Access API）。
- `settings_dir` 周りの扱いを堅牢化
  - UI で `settings_dir` にファイルパスを入力した場合、自動で親フォルダに正規化して扱うようにした（ユーザー入力の柔軟性向上）。
  - ローカルランチャーの `POST /settings` は指定ディレクトリが存在しない場合にフォルダを自動作成して保存するようになった（`qgis_launcher/src/main.rs`）。
- UI の整理
  - いくつかのボタン（「パスに保存」「パスから読込」「フォルダを開く」）を整理・削除し、ファイル読み書きはブラウザ経由とランチャー経由の安定した流れに統一。
- ランチャー関連
  - `qgis_launcher` に `POST /open-folder` を追加（フォルダをエクスプローラで開く挙動）。UI側では現在非表示にしているがランチャーは該当エンドポイントを提供。
